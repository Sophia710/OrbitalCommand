## 1. 产品概述

### 1.1 产品定位

**卫星互联网超级数字员工**是一款面向卫星运营商的智能化测试与运营平台，基于 L4 高度自主智能体技术，将领域专业知识、自动化工具与可审计执行链路深度融合，赋能卫星互联网的测试、验收、运维全流程。

产品形态为**私有化部署的 Web 应用**，核心由"内置专业领域员工 + 通用员工 + 自定义创建平台"三层组成，将重复、复杂、高风险的卫星互联网业务任务，交给可观察、可审计、可回滚的 AI 数字员工执行。

### 1.2 目标用户

- **主要用户**：卫星运营商内部的运营、运维、测控、研发、测试工程师
- **次要用户**：第三方测试机构、外包运维团队（受邀访问受限功能）
- **决策方**：卫星运营商信息化/数字化部门主管、运维部门负责人

### 1.3 核心价值主张

| 价值点 | 描述 |
|--------|------|
| **降低专业门槛** | 业务人员通过自然语言与拖拽即可驱动专业测试与运维任务 |
| **提升闭环效率** | L4 自主执行 + 白盒推理，将测试与运维任务从"小时级"压缩到"分钟级" |
| **沉淀领域知识** | 知识库统一管理 3GPP/ETSI 标准、企业 SOP、设备资料、历史案例 |
| **保障可审计性** | 完整的执行链路追踪、决策依据记录、回滚能力，满足合规要求 |
| **支持灵活扩展** | 零代码拖拽式自定义平台，业务人员可按需创建新数字员工 |

### 1.4 核心术语定义

- **数字员工（Digital Employee）**：基于 LLM 的智能体，可感知任务、调用工具、规划步骤、执行并汇报
- **智能体（Agent）**：具备自主规划、记忆、工具调用与反思能力的 LLM 增强实体，是数字员工的"内核"
- **Skill（技能）**：智能体可调用的原子能力单元（如"查询卫星遥测"、"生成测试报告"）
- **知识库（Knowledge Base）**：智能体检索的领域知识集合，支持文档、FAQ、向量检索
- **插件（Plugin）**：面向特定业务域（如 TTC、网关）的封装模块，可被智能体加载
- **全链路（Full-Chain）**：覆盖用户终端 → 无线链路 → 卫星载荷 → 星地网络 → 地面系统的端到端测试
- **白盒推理（White-box Reasoning）**：智能体执行过程透明化展示，包括思考、规划、工具调用、依据引用均可追溯
- **记忆（Memory）**：智能体的短期对话上下文与长期任务/事件记忆，支持跨任务、跨会话的状态延续

### 1.5 产品愿景

成为卫星互联网行业的"AI 原生操作系统"——让每一位卫星互联网从业者，都拥有专属的"超级数字员工"。

### 1.6 产品范围说明

**包含范围**：
- 内置 4 类专业领域数字员工（用户终端测试 / 星地网络测试 / 卫星载荷测试 / 全链路验收与运维测试）
- 内置 3 类通用数字员工（任务编排 / 文档撰写 / 数据分析）
- 自定义数字员工创建平台（智能体配置、Skill 管理、知识库管理、可视化定制）
- 集成 5 大类外部系统（TTC、OSS/BSS、用户终端管理、载荷测试装备、专项测试装备）
- 完整的安全、审计、运营支撑

**不包含范围**（V1.0）：
- 移动端 APP 与桌面客户端（仅 Web 端）
- 多租户 SaaS 服务（V1.0 仅支持单租户私有化部署）
- 跨卫星运营商的联邦能力
- 实时毫秒级控制（V1.0 仅支持分钟级响应）

## 2. 用户角色与权限

### 2.1 角色定义

系统采用"**管理员 + 普通用户**"两级角色模型，简单清晰，符合运营商实际管理诉求。

#### 2.1.1 管理员（Admin）

- **职责**：平台配置、用户授权、模型管理、数字员工发布审核、知识库基线维护
- **人数限制**：建议不超过 5 人（最小特权原则）
- **典型场景**：开通/关闭账号、上传基线知识库、审核用户提交的自定义数字员工、配置 LLM 推理参数

#### 2.1.2 普通用户（User）

- **职责**：使用平台内置数字员工完成日常任务、按需创建/管理"个人专属"数字员工
- **人数规模**：覆盖全单位业务人员，无上限
- **典型场景**：调用"用户终端测试员工"执行测试任务、创建"我自己的运维助理"、上传自己的私有知识库

### 2.2 权限矩阵

| 功能模块 | 管理员 | 普通用户 |
|---------|--------|---------|
| 登录/个人设置 | ✓ | ✓ |
| 员工广场浏览 | ✓ | ✓ |
| 雇佣（使用）内置数字员工 | ✓ | ✓ |
| 雇佣（使用）其他用户的公开数字员工 | ✓ | ✓（按可见范围） |
| 创建自定义数字员工 | ✓ | ✓ |
| 发布数字员工到员工广场 | ✓（直接发布） | ✓（**需管理员审核**） |
| 上传知识库 | ✓（基线库） | ✓（个人库） |
| Skill 库管理（创建/发布/删除） | ✓ | ✓（个人 Skill 发布需审核） |
| 用户管理（增删改、角色分配） | ✓ | ✗ |
| 模型管理（部署/升级/参数） | ✓ | ✗ |
| 审核中心（处理待审申请） | ✓ | ✗ |
| 审计日志查看 | ✓ | ✓（仅自己） |
| 系统配置（限速、并发、安全） | ✓ | ✗ |
| 主题切换（仅明/暗） | ✓ | ✓（仅个人） |

### 2.3 可见性规则

- **内置数字员工**：所有人可见可用
- **已发布的数字员工**：所有人可见可用
- **待审核中的员工**：仅创建者可见
- **被驳回的员工**：仅创建者可见，并显示驳回原因
- **草稿态员工**：仅创建者可见
- **个人知识库**：仅创建者本人及管理员可见

### 2.4 鉴权与登录

- 支持账号密码登录、SSO（OAuth 2.0 / SAML 2.0，可对接企业统一身份源）
- 支持 MFA（多因素认证），管理员强制开启
- 细粒度操作日志：所有写操作、敏感读操作均记录至审计日志，保留期 ≥ 3 年

### 2.5 审核流程

**普通用户发布数字员工流程**：

1. 用户在创建平台完成数字员工设计与测试
2. 提交"发布到员工广场"申请
3. 进入"审核中心"待办列表，管理员收到通知
4. 管理员可查看配置详情、知识库内容、Skill 清单、测试用例与试运行结果
5. **通过** → 数字员工进入员工广场，所有人可使用
6. **驳回** → 退回用户，附驳回原因，用户可修改后重新提交
7. **审核 SLA**：管理员需在 2 个工作日内处理；超时自动提醒，超 5 个工作日自动升级上级管理员

## 3. 核心功能 - 内置专业领域数字员工

> 本章覆盖 4 类内置专业领域数字员工。每类员工均采用"**领域插件 + 通用智能体运行时**"的架构模式，由管理员在基线阶段构建并发布到员工广场。

### 3.0 通用架构

每类专业员工由 4 部分组成：

1. **智能体配置（Persona & Prompt）**：身份、目标、能力边界、行为约束
2. **领域 Skill 库**：该员工专属的可调用技能
3. **领域知识库**：该员工专属的检索知识
4. **领域插件（Domain Plugin）**：对接该业务域的连接器集合

所有专业员工共享同一智能体运行时，确保一致性、可观测性与可升级性。

### 3.1 用户终端智能化测试数字员工

#### 3.1.1 目标场景

覆盖用户终端（CPE、模组、终端芯片）的功能、性能、协议一致性、OTA 升级等测试任务。

#### 3.1.2 核心能力

- **测试方案生成**：根据测试目标（如"验证 CPE 在 Ka 频段吞吐量"），自动生成测试方案（参数、工装、步骤、判定标准）
- **测试用例执行**：调用终端管理平台、测试仪表、链路仿真装备，按测试序列自动执行
- **结果分析**：解析测试日志、原始数据、性能指标（吞吐量、时延、EVM、误块率），自动判定 Pass/Fail
- **报告生成**：输出标准化测试报告（含环境信息、步骤回放、原始数据、图表、结论）
- **OTA 升级验证**：自动验证终端固件/软件的远程升级流程、回归测试
- **批量回归**：对多型号/多版本终端执行回归测试集

#### 3.1.3 典型 Skill（示例）

- 查询终端台账与配置
- 触发链路仿真
- 抓取与分析终端 log
- 拉取 OTA 升级任务状态
- 生成并下发测试任务工单
- 解析测试结果文件

#### 3.1.4 知识库内容

3GPP TS 38 系列（终端一致性）、ETSI EN 303 687、终端设备手册、典型测试场景库、典型问题案例库

### 3.2 星地网络智能化测试数字员工

#### 3.2.1 目标场景

覆盖卫星与地面网络（含信关站、核心网、地面传输）的端到端测试、协议验证、性能基准。

#### 3.2.2 核心能力

- **网络拓扑探查**：自动发现信关站、地面传输链路、核心网网元
- **协议一致性测试**：执行 GTP、PMIP、TCP/IP、QoS 等协议栈测试
- **性能基准测试**：吞吐量、抖动、丢包率、切换时延等指标采集
- **故障注入与回放**：通过仿真装备模拟链路抖动、节点故障，验证系统韧性
- **网络变更验证**：地面段配置变更后，自动验证业务影响
- **报告与告警**：输出网络健康度评分、异常告警

#### 3.2.3 典型 Skill

- 调用网管 OSS 接口
- 拉取性能计数器
- 下发仿真流量
- 解析协议抓包
- 触发网元告警
- 执行网元配置回滚

#### 3.2.4 知识库内容

3GPP TS 23.501/23.502（系统架构）、ETSI TS 103 786、信关站运维手册、典型故障处理预案、协议一致性测试规范

### 3.3 卫星载荷智能化测试数字员工

#### 3.3.1 目标场景

覆盖通信卫星载荷（透明转发/星上处理）的功能、性能、接口、在轨测试验证。

#### 3.3.2 核心能力

- **载荷状态采集**：自动汇总遥测数据（温度、功耗、功放状态、波束指向）
- **载荷性能测试**：EIRP、G/T、频率切换、波束跳变等指标测试
- **接口验证**：验证载荷与数管、电源、姿控、热控、测控的接口数据正确性
- **测试序列编排**：根据测试目标自动编排长时序、多步骤的测试任务
- **异常诊断**：结合遥测趋势与历史基线，识别载荷异常征兆
- **测试装备集成**：控制测试仪表、仿真器、模拟源

#### 3.3.3 典型 Skill

- 拉取载荷遥测
- 下发遥控指令（受审批）
- 调用测试仪表（信号源、频谱仪）
- 执行在轨测试序列
- 解析 CCSDS 帧
- 载荷健康度评估

#### 3.3.4 知识库内容

CCSDS 131.0-B、CCSDS 232.0-B、卫星平台手册、载荷技术规格书、典型异常案例库

### 3.4 全链路智能化验收与运维测试数字员工

#### 3.4.1 目标场景

覆盖"用户终端 ↔ 无线链路 ↔ 卫星载荷 ↔ 星地网络 ↔ 业务平台"端到端验收、回归、运维测试。

#### 3.4.2 核心能力

- **端到端测试编排**：跨域（终端+网络+载荷）测试任务统一编排与执行
- **跨域数据关联**：将多域数据（终端 log + 网管性能 + 载荷遥测）按时间戳关联分析
- **验收用例库管理**：维护整星/整网验收用例库，支持版本管理与回归
- **故障定位与根因分析**：跨域异常时自动定位可疑域并提供处置建议
- **运维剧本执行**：将 SOP 编排为可执行剧本，一键执行或定时触发
- **运维报告自动化**：周期性生成全链路健康度报告、可用性报告、SLA 报告

#### 3.4.3 典型 Skill

- 调用其他 3 个专业员工的 Skill（编排型）
- 跨域数据时间对齐
- 触发跨域告警
- 生成 SLA 报告
- 执行运维剧本
- 跨域故障关联分析

#### 3.4.4 知识库内容

- 跨域关联规则库
- 端到端验收标准（如系统级验收大纲）
- 故障处置剧本库
- SLA/可用性计算规则
- 历史全链路故障复盘

### 3.5 专业员工共性要求

- **白盒执行**：每一步推理、规划、工具调用均对用户透明可见
- **回滚能力**：对执行类操作提供"撤销/补偿"接口
- **审批机制**：高风险操作（如远程下发遥控指令）必须人工审批
- **可观测性**：所有执行链路写入审计日志，支持事后追溯与回放
- **可升级**：每个员工配置版本化，支持灰度发布与回滚
- **可测试**：内置"试运行"沙箱，可输入样例任务验证配置正确性

## 4. 核心功能 - 内置通用数字员工

> 通用数字员工与专业员工共享同一智能体运行时，差异在于：通用员工不绑定特定业务域，能力更通用、适用更广，可作为"通用助理"被所有用户直接雇佣使用。

### 4.1 任务编排与流程助理（Task Orchestrator）

#### 4.1.1 目标场景

将多步骤、跨系统的复杂任务自动化串联，充当"任务指挥官"角色。

#### 4.1.2 核心能力

- **多步骤任务规划**：理解用户的复杂意图，自动拆解为子任务并规划执行顺序
- **跨员工调度**：可调用多个专业数字员工的 Skill 完成复杂任务（如"先做终端测试，再做网络验证，最后生成报告"）
- **条件分支与循环**：根据中间结果决定下一步动作
- **定时与触发器**：支持定时执行、事件触发（如"每天早上 9 点生成全链路健康报告"）
- **异常处理**：执行失败时支持重试、人工介入、跳过、降级
- **执行大屏**：提供甘特图式执行链路可视化

#### 4.1.3 典型 Skill

- 调用其他员工的 Skill
- 定时任务调度
- 条件判断节点
- 邮件/IM 通知
- 审批节点
- 任务编排节点

#### 4.1.4 知识库内容

企业内部 SOP 索引、典型工作流模板库

### 4.2 文档撰写与报告生成员（Document & Report Specialist）

#### 4.2.1 目标场景

作为"专职写手"，快速产出各类文档、报告、纪要、邮件。

#### 4.2.2 核心能力

- **多类型文档生成**：测试报告、运维报告、故障分析报告、会议纪要、邮件、工作总结
- **模板化输出**：管理员可预置文档模板，员工可调用模板生成符合规范的文档
- **数据可视化**：自动从结构化数据生成图表（折线、柱状、饼图、热力图、拓扑图）
- **多源信息整合**：将来自数据库、文档、对话的数据融合到一份报告
- **多版本管理**：保留历史版本，支持对比
- **导出格式**：PDF、Word、Markdown、HTML

#### 4.2.3 典型 Skill

- 读取结构化数据（数据库、CSV、JSON）
- 生成图表
- 套用模板
- 导出 PDF/Word
- 翻译（多语种）
- 内容校对

#### 4.2.4 知识库内容

企业内部报告模板库、写作风格指南、术语表

### 4.3 数据分析与决策辅助员（Data Analyst）

#### 4.3.1 目标场景

为业务决策提供数据支持，覆盖指标计算、趋势分析、异常检测、归因分析。

#### 4.3.2 核心能力

- **自然语言查询**：用户用自然语言提问（如"上个月链路可用率多少？"），自动生成 SQL/查询语句
- **指标计算**：吞吐量、可用率、SLA、MTTR 等指标自动计算
- **趋势分析**：同比、环比、季节性、长期趋势
- **异常检测**：基于统计/机器学习方法识别异常波动
- **归因分析**：指标异常时自动分析可能原因（如"链路可用率下降 3%，主要因信关站 2 节点 8 月 12 日切换失败"）
- **可视化看板**：将分析结果以交互式看板呈现
- **预测分析**：基于历史数据做短期预测（如"未来 7 天可用率预测"）

#### 4.3.3 典型 Skill

- 自然语言转 SQL
- 执行查询
- 统计建模
- 异常检测算法调用
- 看板渲染
- 指标计算引擎

#### 4.3.4 知识库内容

- 数据字典（业务术语表）
- 指标口径说明
- 常见分析模板
- 业务基线数据

### 4.4 通用员工共性要求

- **零配置可用**：用户雇佣后即可使用，无需复杂配置
- **任务历史**：所有任务执行历史可追溯、可重放
- **协同能力**：通用员工可与专业员工协作（在任务编排中作为节点）
- **可个性化**：用户可上传个人偏好（如"我喜欢简洁风格"、"我的报告包含这些章节"）
- **可学习**：基于用户反馈持续优化（管理员可关闭/开启此能力）

## 5. 核心功能 - 自定义数字员工创建平台

> 自定义平台是产品的"创作工具"，让业务人员无需写代码即可创建专属数字员工。平台采用**零代码拖拽式**设计，由智能体配置、Skill 创建与管理、知识库管理、可视化定制四大功能模块组成。

### 5.1 智能体配置（Agent Configuration）

#### 5.1.1 基础属性配置

- **员工名称、头像、简介**：展示在员工广场
- **员工类型**：单选（专业领域/通用）
- **领域标签**：多选（如"测试"、"运维"、"数据分析"）
- **目标用户**：多选（管理员/普通用户/所有人）

#### 5.1.2 智能体核心配置

- **身份（Persona）**：设定身份描述（如"我是张三，一名专注 Ka 频段测试的工程师"）
- **目标（Goal）**：核心职责描述
- **能力边界**：明确能做什么、不能做什么（如"不主动修改系统配置，必须人工确认"）
- **行为约束**：禁止行为清单（如"绝不向外部网络发送数据"）
- **对话风格**：正式/轻松/技术型/商务型 等
- **语言偏好**：中文为主，可扩展多语

#### 5.1.3 高级配置

- **模型选择**：可选用不同尺寸的开源模型（7B/14B/32B/70B）
- **Temperature/Top-p 等推理参数**
- **最大思考步数、最大工具调用次数**：防失控
- **记忆策略**：短期/长期记忆开关、记忆清理策略
- **审批节点配置**：哪些操作必须人工审批

### 5.2 Skill 技能组件管理

#### 5.2.1 Skill 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **系统内置 Skill** | 平台预置，公共可用 | "HTTP 请求"、"数据库查询"、"文件读取" |
| **领域插件 Skill** | 来自领域插件（4 类专业员工） | "查询卫星遥测"、"触发 OTA 升级" |
| **用户自定义 Skill** | 业务人员创建（个人/公开） | "查询某内网接口"、"格式化我的报告" |

#### 5.2.2 自定义 Skill 创建流程

1. **基本信息**：名称、描述、分类、图标
2. **输入输出定义**：参数列表（含类型、必填、说明）、返回结构
3. **执行逻辑配置**（**核心**，零代码）：
   - 简单型：填写 HTTP API（URL、方法、Header、参数映射）
   - 脚本型：上传 Python/Shell 脚本（沙箱执行）
   - 组合型：拖拽编排多个子步骤
4. **测试验证**：在沙箱中输入样例数据验证 Skill 是否正确
5. **版本管理**：保存历史版本，可回滚
6. **发布**：可发布为"个人 Skill"或提交"公开 Skill"申请（需管理员审核）

#### 5.2.3 Skill 库浏览与检索

- 按分类、标签、来源（系统/领域/用户）、使用次数浏览
- 全文搜索（按名称、描述、参数）
- 详情页：含文档、调用示例、版本历史、调用统计

### 5.3 知识库管理

#### 5.3.1 知识库分类

- **个人知识库**：仅本人可见
- **公共知识库**：管理员审核后所有人可见
- **基线知识库**：管理员维护，作为产品出厂基线

#### 5.3.2 知识来源支持

- **文件上传**：PDF、Word、Excel、PPT、TXT、Markdown
- **数据库对接**：PostgreSQL、MySQL、Oracle、达梦、人大金仓等
- **API 接入**：自定义 HTTP API 作为知识源
- **手工录入**：直接录入 FAQ、术语表、流程文档
- **历史会话导入**：将用户与数字员工的历史对话中有价值的问答沉淀为知识条目

#### 5.3.3 知识处理流水线

1. **解析**：多格式文档解析、表格识别、OCR（图片）
2. **分块（Chunking）**：按语义/段落/自定义规则切分
3. **向量化**：使用 Embedding 模型生成向量（默认使用 BGE/M3E 等开源模型，可选远程 API）
4. **索引构建**：写入向量数据库（默认 Milvus/PGVector，可选国产化）
5. **元数据标注**：自动/手工标注分类、有效期、敏感等级

#### 5.3.4 知识检索能力

- **语义检索**：基于向量相似度
- **关键词检索**：BM25 等传统检索
- **混合检索**：语义+关键词融合排序
- **结构化查询**：对元数据过滤（"查 2024 年关于 Ka 频段的文档"）
- **多知识库联合检索**：跨库检索、统一排序

#### 5.3.5 知识库维护

- **生命周期管理**：版本、更新、过期、归档
- **质量评估**：命中率、引用准确率
- **去重与合并**：相似内容自动合并建议
- **权限管理**：细粒度到文档/段落级访问控制（敏感文档）

### 5.4 可视化定制（Visual Customization）

#### 5.4.1 员工"画布"

每个数字员工有一个独立的"画布"，可视化展示与编辑该员工的全部组成：

- **中部**：智能体配置（基础属性 + 高级参数）
- **左侧**：Skill 树（按层级组织该员工可调用的所有 Skill）
- **右侧**：知识库（关联的知识库清单）
- **底部**：测试与试运行区

#### 5.4.2 拖拽式工作流编排

- 用户可拖拽节点（输入、LLM、Skill 调用、条件判断、循环、审批、输出）到画布
- 节点间用连线表达执行顺序与数据流
- 支持变量绑定（如将"用户输入"绑定到 Skill 的"query"参数）
- 内置节点类型：
  - 触发节点（用户输入、定时器、Webhook）
  - LLM 节点（调用大模型，支持 Prompt 模板）
  - Skill 节点（调用任一 Skill）
  - 逻辑节点（条件、循环、并行、合并）
  - 工具节点（HTTP、SQL、文件、邮件）
  - 审批节点（人工介入）
  - 输出节点（返回结果、写入存储、发送通知）

#### 5.4.3 调试与试运行

- **逐节点调试**：可单步执行每个节点
- **变量监视**：实时查看每个节点输入/输出
- **样例任务库**：内置测试用例，验证员工行为
- **回归测试**：修改后跑历史用例集，确保不退化

#### 5.4.4 模板与示例

- 平台预置多个**模板员工**（如"日报生成助理"、"会议纪要助理"、"告警摘要助理"）
- 用户可基于模板"复制-修改-发布"快速创建

### 5.5 平台核心体验指标

- **从想法到可用员工的时间**：通过模板 < 30 分钟；纯自定义 < 4 小时
- **拖拽创建工作流的可用性**：业务人员经 1 小时培训即可上手
- **试运行-发布闭环**：用户在平台内即可完成创建-测试-发布全流程

## 6. 界面原型描述

> 产品的视觉风格延续参考截图风格：左侧主导航 + 顶部用户区 + 主内容区。本章描述关键页面的布局与核心交互。

### 6.1 全局导航结构

**左侧主导航**（4 项，简洁直接）：

- **员工广场**：浏览并雇佣平台提供的数字员工
- **我的员工**：管理已雇佣/自己创建的所有数字员工
- **创建员工**：进入数字员工创建平台（智能体配置、Skill、知识库、画布）
- **数据应用**：数据看板、自助分析入口

**顶部用户区**（右侧）：通知、帮助、主题切换（明/暗）、设置、退出

> **入口规则**：取消右下角浮动"快速新建对话"按钮。**新建对话的唯一路径**：进入"我的员工" → 点击具体数字员工 → 在对话页左侧"+"按钮新建会话。数据应用模块提供独立的分析对话入口。

**主题切换（仅明暗两种模式）**：

- 位置：顶部用户区（右侧），位于"帮助"和"设置"之间
- 形态：浅色主题时显示 ☀（太阳）图标；深色主题时显示 🌙（月亮）图标
- 交互：
  - 点击 ☀/🌙 图标 → 一键切换浅色/深色主题（即时生效，全局一致）
  - 切换过程平滑过渡：颜色、背景、边框、阴影统一 220 ms 缓动；首次加载无过渡，避免白闪
  - 用户偏好持久化：用户的选择保存到本地，刷新后自动恢复
  - **明确说明：本系统仅提供明（浅色）、暗（深色）两种主题模式，不提供主色调色板或品牌色切换功能**；强调色（默认紫罗兰色系）作为单一品牌主色贯穿全站

### 6.2 关键页面

#### 6.2.1 员工广场

搜索栏（全部 / 数字员工 / 关键词搜索）+ 右上角"一键雇佣数字员工"+ 员工卡片网格（头像、名称、标签、简介、立即雇佣/已雇佣/联系定制按钮）

#### 6.2.2 我的员工

分类标签（全部 / 内容运营 / 产品顾问 / 业务调整 / 通用个人助理 / 全部状态）+ 搜索 + "新增员工"按钮 + 员工卡片列表（头像、名称、状态、最后活跃时间、操作）

**交互**：点击员工卡片进入该员工的"对话主页"

#### 6.2.3 员工对话页

- **底部**：输入区（文字、文件、图片、语音）

**对话交互**：

- 输入支持文字、文件、图片、语音
- 发送后实时显示思考过程（白盒推理）
- 工具调用卡片：展示员工正在调用哪个 Skill、参数、结果
- 审批节点：高风险操作弹出审批卡片

- 引用来源可悬停查看原文

#### 6.2.4 数字员工创建平台 - 智能体配置页

**布局**：左右双栏 + 顶部预览

- 甘特图：展示复杂任务（含子任务）的执行进度、时间线、状态（运行中/完成/失败/等待审批）

**布局**：筛选器 + 表格 + 详情

- 筛选：时间范围、用户、操作类型、对象、结果
- 表格：时间、用户、操作、对象、结果、IP
- 详情：操作前后快照、调用链

- 分享：点击"分享"生成链接，可设置有效期（1天/7天/30天/永久）、访问密码、下载次数限制

### 6.3 设计规范

- **品牌主色**（强调色，单一固定值，**不支持用户切换**）：
  - 默认品牌色：紫罗兰（Violet `#8b5cf6` → `#d946ef` 渐变），作为全站统一强调色
  - 用于：主操作按钮、激活态、关键图表、聚焦态、品牌资产、图标高亮
- **主题**（明/暗两套，受用户控制）：
  - **浅色主题**：浅紫/粉色调（与参考截图一致），传达"亲和、专业、智能"
  - **深色主题**：深色背景 + 高对比强调色，适合长时间使用、运维值班、指挥中心
- **主题切换**：
  - 用户可一键切换浅色/深色主题，切换过程平滑（220 ms 颜色/背景/边框/阴影统一过渡）
  - 切换时全局 UI 一致变化：背景色、表面层、文本层级、边框、阴影、cosmos 背景网格等所有视觉属性同步切换
  - 首次加载无过渡动画（避免白闪）；切换瞬间启用过渡
  - 用户偏好持久化至本地，刷新后自动恢复
  - **明确约束：系统不提供主色调色板、不提供品牌色切换、不提供多主题色方案**
- **字体**：思源黑体（中文）/ Inter（英文）
- **响应式**：最低支持 1366×768，主要面向桌面浏览器
- **浏览器兼容**：Chrome 100+、Edge 100+、Firefox 90+
- **可访问性**：遵循 WCAG 2.1 AA 级标准

## 7. 数据需求

### 7.1 数据分类

| 数据类别 | 描述 | 存储要求 |
|---------|------|---------|
| **用户与权限数据** | 用户账号、角色、组织关系 | 关系型数据库 |

| **审计日志数据** | 所有写操作、敏感读操作记录 | 关系型 + 冷存储 |
| **知识库内容** | 文档原文、分块、向量、元数据 | 对象存储 + 向量库 |
| **模型与推理数据** | 模型配置、推理缓存、性能指标 | 关系型 + 时序库 |
| **业务数据缓存** | 调用外部系统的临时缓存 | 关系型 |
| **系统运行数据** | 服务健康、性能指标、错误日志 | 时序库 + 日志库 |

### 7.2 关键实体（核心模型）

#### 7.2.1 用户与权限

- `User`：用户（id、姓名、账号、邮箱、角色、组织、状态、创建时间、MFA 状态）
- `Role`：角色（id、名称、描述、权限集合）
- `Permission`：权限点（id、名称、资源、操作）
- `Organization`：组织（id、名称、父组织、层级）
- `Session`：会话（id、用户、token、IP、UA、过期时间）

#### 7.2.2 数字员工

- `DigitalEmployee`：员工（id、名称、头像、类型、创建者、可见性、状态、版本号、发布时间）
- `AgentConfig`：智能体配置（id、员工id、persona、goal、约束、风格、模型id、参数 JSON）
- `Skill`：技能（id、名称、描述、分类、来源、版本、执行逻辑、输入输出 schema、状态）
- `SkillVersion`：技能版本（id、skill_id、版本号、变更说明、发布时间）
- `Workflow`：工作流（id、员工id、节点、连线、变量定义、版本号）
- `KnowledgeBase`：知识库（id、名称、分类、可见性、创建者、Embedding 模型、检索配置）
- `Document`：文档（id、知识库id、文件名、类型、大小、上传时间、状态、向量库 id）
- `Chunk`：分块（id、文档id、序号、原文、向量 id、元数据）

#### 7.2.3 对话与任务

- `Conversation`：会话（id、员工id、用户id、标题、创建时间、最后消息时间、消息数）
- `Message`：消息（id、会话id、角色、内容、引用、工具调用、思考过程、时间、token 消耗）
- `Task`：任务（id、会话id、员工id、类型、状态、起止时间、父任务id、当前节点）
- `TaskStep`：任务步骤（id、任务id、节点id、状态、起止时间、输入输出、错误信息）
- `ApprovalRequest`：审批请求（id、任务id、节点id、申请人、审批人、内容、状态、时间）

#### 7.2.4 审计

- `AuditLog`：审计日志（id、用户、操作、资源、对象、结果、IP、UA、时间、前后快照）
- `ApiCallLog`：API 调用日志（id、用户、API、参数、响应、耗时、状态）

#### 7.2.5 模型与系统

- `Model`：模型（id、名称、类型、版本、路径、参数、状态、性能指标）
- `SystemConfig`：系统配置（id、键、值、描述、修改人、时间）
- `PromptTemplate`：Prompt 模板（id、名称、内容、版本、分类）

#### 7.2.6 文件与产出物

- `File`：文件（id、文件名、类型、大小、所属员工、所属会话、所属任务、上传者、上传时间、存储路径、状态、版本号、敏感等级、分享设置）
- `FileShare`：文件分享（id、文件id、分享链接、有效期、密码、下载次数限制、创建时间、状态）

### 7.3 知识图谱（领域本体）

为支撑卫星互联网专业员工的高质量回答，建议构建如下领域本体：

- **核心实体**：卫星、载荷、信关站、地面站、用户终端、波束、链路、协议
- **关系**：卫星-包含-载荷、信关站-服务-波束、终端-接入-波束、链路-基于-协议
- **属性**：卫星（编号、轨道、频段、平台）、载荷（类型、EIRP、G/T、波束数）、信关站（位置、频段、天线尺寸）、终端（型号、频段、协议栈）
- **事件**：发射、入网、切换、故障、维护、退役
- **指标**：吞吐量、时延、可用率、EIRP、G/T、误码率

本体数据用于：

- 增强知识库检索（实体链接）
- 增强推理（实体关系推理）
- 跨员工协同（统一术语）

### 7.4 向量库设计

- **默认引擎**：Milvus（单节点起步，集群扩展）
- **国产化可选**：支持替换为达梦向量、PGVector
- **Embedding 模型**：BGE-M3 / M3E（中文优化）
- **分块策略**：默认按 512 token + 50 token 重叠，特殊文档类型（表格、代码）可自定义
- **元数据**：分类、来源、敏感等级、有效期、所属员工/知识库
- **多租户隔离**：通过集合/分区隔离不同部门/员工的向量数据

### 7.5 数据生命周期与合规

- **保留期**：
  - 对话与执行数据：在线 1 年 + 冷存储 3 年
  - 审计日志：在线 3 年 + 冷存储 7 年
  - 业务数据：按业务需求
  - 临时缓存：按需
- **数据销毁**：用户/管理员删除数据时，30 天后从冷存储彻底清除
- **数据加密**：传输 TLS 1.3；存储 AES-256
- **数据脱敏**：对外输出时自动脱敏（如手机号、密钥、机密参数）
- **数据分类**：按敏感等级分类（公开、内部、秘密、机密），不同等级不同保护策略
- **合规要求**：符合《数据安全法》《个人信息保护法》及卫星行业相关规定

### 7.6 数据规模预估（首年）

- 用户：500 人
- 数字员工：内置 7 个（4 专业 + 3 通用） + 用户创建 200+ 个
- 对话：日均 5,000 轮
- 知识库：100 个库，文档总量 50,000 份
- 向量数据：~1 亿条分块
- 审计日志：日均 100,000 条
- 模型推理：日均 10,000 次

## 8. 业务规则与工作流

### 8.1 核心业务规则

#### 8.1.1 数字员工生命周期规则

- **创建态 → 草稿态**：保存配置
- **草稿态 → 测试态**：试运行通过
- **测试态 → 审核态**：提交发布申请
- **审核态 → 发布态**：管理员通过
- **发布态 → 灰度态**：可配置灰度比例（10%/50%/100%）
- **灰度态 → 正式态**：灰度无问题自动全量
- **发布态 → 归档态**：管理员主动归档或长期未使用自动归档

#### 8.1.2 任务执行规则

- **高风险操作必须人工审批**：如"下发遥控指令"、"修改业务配置"、"触发 OTA 升级"
- **审批 SLA**：普通任务 5 分钟内，紧急任务 1 分钟内响应（推送通知）
- **超时处理**：审批超时自动按规则处理（默认拒绝，可配置自动通过）
- **执行失败重试**：默认重试 2 次，可配置
- **回滚**：执行类操作提供撤销/补偿接口

#### 8.1.3 知识库规则

- **基线知识库**只能由管理员修改
- **个人知识库**仅创建者可见、可编辑、可删除
- **公开知识库**发布需管理员审核
- **版本管理**：所有修改产生新版本，可回滚
- **过期策略**：可设置文档有效期，过期前提醒更新

#### 8.1.4 模型与推理规则

- **模型升级**：管理员操作，先灰度后全量
- **推理限速**：单用户 100 RPM、平台 1000 RPM（可配）
- **并发限制**：单用户 5 个任务、平台 100 个任务（可配）
- **Token 配额**：单用户/日有上限（可配）

### 8.2 典型工作流

#### 8.2.1 工作流 1：用户终端入场测试

**参与员工**：用户终端智能化测试数字员工

**流程**：

1. 测试工程师在"我的员工"中选择"用户终端测试员工"，进入对话
2. 输入任务："对 50 台 CPE 进行 Ka 频段吞吐量测试"
3. 员工自动：
   - 解析需求（识别测试目标、批次、设备类型）
   - 生成测试方案（含参数、工装、判定标准）
   - 向工程师展示方案，等待确认
4. 工程师确认，员工调度执行
5. 员工调用"链路仿真 Skill"、"终端管理 Skill"执行测试
6. 实时返回测试进度
7. 测试完成，自动分析结果，生成报告
8. 工程师可在对话中追问、要求重测、归档报告

#### 8.2.2 工作流 2：全链路故障定位

**参与员工**：全链路智能化验收与运维测试数字员工

**流程**：

1. 运维人员发现某区域可用率异常
2. 启用"全链路员工"对话，输入："分析过去 24 小时华北区域可用率下降原因"
3. 员工自动：
   - 调取终端、网络、载荷三方数据
   - 时间对齐后进行跨域关联分析
   - 调用历史案例库匹配类似故障
4. 输出"根因候选清单"（如"信关站 2 节点切换失败 5 次，疑似主因"）
5. 提供处置建议（如"建议执行切换剧本，重启服务"）
6. 运维人员选择执行处置，员工按剧本自动执行（含审批节点）
7. 执行完成后输出故障复盘报告

#### 8.2.3 工作流 3：定时运维报告生成

**参与员工**：任务编排助理 + 文档撰写员 + 数据分析员

**流程**：

1. 管理员在"任务编排助理"中创建定时任务："每周一 9:00 生成全链路周报"
2. 配置任务节点：
   - 触发：定时器
   - 节点 1：调用"数据分析员"统计上周关键指标
   - 节点 2：调用"全链路员工"生成健康度评分
   - 节点 3：调用"文档撰写员"按模板生成周报
   - 节点 4：邮件发送给指定收件人
3. 任务发布，员工按计划自动执行
4. 执行大屏可查看每次执行的甘特图与详情
5. 报告归档到知识库，可被检索

#### 8.2.4 工作流 4：用户自定义数字员工

**参与员工**：用户本人

**流程**：

1. 用户在"创建员工"中点击"基于模板新建"或"从零新建"
2. 选择模板（如"日报生成助理"），进入画布
3. 修改基础属性（名称、头像、简介）
4. 修改智能体配置（Persona、目标、约束）
5. 调整工作流（如增加"调用我的接口"节点）
6. 关联个人知识库
7. 试运行：输入"生成本周工作日报"
8. 检查输出，满意后保存草稿
9. 点击"发布到员工广场"，提交审核
10. 管理员审核 → 通过 → 全员可见可用

### 8.3 人机协作模式

#### 8.3.1 三种协作模式

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| **人类主导** | 数字员工只做建议，最终决策与执行由人完成 | 高风险、首次执行、复杂决策 |
| **人机协同** | 数字员工执行，每步结果需人确认 | 中等风险、有标准流程 |
| **机器主导** | 数字员工独立执行，仅在异常时通知人 | 低风险、高频、标准化任务 |

#### 8.3.2 异常升级机制

- 数字员工遇到超出能力范围的情况，自动暂停并通知相关人员
- 通知渠道：站内消息、邮件（可扩展 IM、短信）
- 升级策略：默认 3 分钟未响应升级至上级，超时可配置

### 8.4 限速与配额

- 单用户任务并发：5 个
- 平台任务并发：100 个
- 单用户 API 调用：100 RPM
- 平台 API 调用：1000 RPM
- 单用户日 Token 上限：默认 1M
- 平台日 Token 上限：默认 100M
- 知识库单库最大文档数：100,000
- 知识库单文档最大：100MB
- 自定义员工单用户最大：50 个
- 工作流单员工最大节点数：200 个
- 会话保留：单用户最多保留 1000 个会话

> 所有限速、配额、保留策略可由管理员在系统配置中调整。

### 8.5 文件输出与管理

#### 8.5.1 文件类型

数字员工在执行任务中可能产出多种类型文件：

- **报告类**：测试报告、运维报告、故障分析报告
- **数据类**：导出的结构化数据（CSV、Excel、JSON）
- **图表类**：可视化图表（PNG、SVG、PDF）
- **日志类**：执行日志、抓取数据、原始数据
- **代码/脚本类**：自动生成的测试脚本、配置脚本
- **中间产物**：工作流执行中的临时文件

#### 8.5.2 文件存储

- **存储位置**：平台统一对象存储
- **存储路径**：`/files/{user_id}/{employee_id}/{conversation_id}/{task_id}/{filename}`
- **存储加密**：AES-256
- **去重策略**：相同内容（哈希）自动去重

#### 8.5.3 文件管理规则

- **归属**：文件归属产生该文件的数字员工与会话
- **可见性**：
  - 用户在自己员工会话中产生的文件：仅自己与管理员可见
  - 公共员工产生并发布到"共享区"的文件：所有用户可见
- **生命周期**：
  - 在线保留 1 年（可配）
  - 冷存储保留 3 年
  - 用户可手动删除（删除后 30 天可恢复，30 天后彻底清除）
- **版本管理**：同名文件新版本保留历史，支持回滚
- **下载/分享**：
  - 默认需本人下载
  - 分享：可生成带过期时间的分享链接（需审批或可配置）
  - 大文件（>100MB）支持分片下载
- **审计**：所有下载、分享、删除操作均记录审计日志

#### 8.5.4 文件管理入口

- 在"我的员工"对话页中，对话详情增加"文件" Tab，展示本次会话所有产出文件
- 提供独立"文件中心"页面（在"我的员工"下作为子页面），跨员工/跨会话管理所有文件
- 文件中心支持：搜索、筛选、批量下载、批量删除、分享

## 9. 集成需求

> 数字员工的"手脚"通过标准化的集成适配层实现。本章描述与 5 大类外部系统的对接规范。

### 9.1 集成总体架构

**集成适配层**位于产品平台与外部业务系统之间，提供：

- **连接器（Connector）**：每个外部系统对应一个连接器，封装协议、鉴权、数据转换
- **数据采集器（Collector）**：定时/实时拉取外部系统数据，缓存至平台
- **触发器（Trigger）**：监听外部系统事件，触发数字员工任务
- **执行代理（Executor）**：将数字员工指令下发到外部系统

**部署模式**：

- 集成适配层与平台同部署
- 连接器以插件形式实现，可热插拔
- 适配器配置由管理员在系统配置中维护

### 9.2 集成清单与规范

#### 9.2.1 卫星测控/TTC 系统

- **对接对象**：测控中心、运控中心
- **对接协议**：CCSDS 131.0、CCSDS 232.0、自定义 TCP/UDP 协议
- **数据能力**：
  - 遥测数据：实时/历史查询、订阅
  - 遥控指令：下发（受严格审批控制）
  - 轨道数据：TLE、星历、状态
  - 卫星状态：健康度、模式、计划
- **触发器**：遥测异常、计划事件、状态变更
- **执行代理**：指令组装、加密、签名、发送、回执确认
- **安全要求**：双向认证、命令签名、完整审计

#### 9.2.2 信关站与地面网络（OSS/BSS/网管）

- **对接对象**：OSS、BSS、网管、核心网、地面传输
- **对接协议**：REST API、SNMP、NetFlow、sFlow、Syslog、SSH/Telnet
- **数据能力**：
  - 性能数据：KPI 实时/历史
  - 配置数据：网元配置、版本
  - 告警：实时告警流
  - 业务数据：用户数、流量、计费
- **触发器**：KPI 越界、告警、配置变更
- **执行代理**：配置下发、配置回滚、命令执行
- **安全要求**：API OAuth 2.0、SNMPv3、SSH 密钥

#### 9.2.3 用户终端管理

- **对接对象**：终端管理平台、OTA 系统、用户业务平台
- **对接协议**：REST API、MQTT、CoAP、WebSocket
- **数据能力**：
  - 终端台账：型号、固件、位置、状态
  - OTA 任务：版本、进度、结果
  - 终端业务：流量、信号质量、用户行为
- **触发器**：OTA 完成、终端告警、批量异常
- **执行代理**：OTA 任务下发、终端远程配置、远程诊断
- **安全要求**：设备证书、链路加密

#### 9.2.4 卫星载荷测试装备

- **对接对象**：测试仪表、仿真器、模拟源、测试管理平台
- **对接协议**：SCPI（仪器控制）、LXI、IVI、REST API、自定义协议
- **数据能力**：
  - 仪表状态与配置
  - 测试任务执行
  - 测试数据回传
  - 测试结果
- **触发器**：测试计划启动、测试结果异常
- **执行代理**：仪表控制指令、测试序列下发、数据采集
- **安全要求**：操作员鉴权、操作记录

#### 9.2.5 专项测试装备（链路仿真、指标分析）

- **对接对象**：链路仿真仪、信道模拟器、协议分析仪、EVM 测试仪
- **对接协议**：SCPI、LXI、MATLAB API、Python SDK、REST API
- **数据能力**：
  - 仿真配置
  - 仿真数据流
  - 指标分析结果
  - 协议解码结果
- **触发器**：仿真任务启动、指标异常
- **执行代理**：仿真任务编排、参数配置、结果回传
- **安全要求**：操作员鉴权

### 9.3 通用集成规范

#### 9.3.1 API 规范

- **平台对外 API**：RESTful + OpenAPI 3.0；鉴权 OAuth 2.0；幂等设计；统一错误码；限流
- **Webhook**：支持入站 Webhook 触发数字员工任务
- **消息队列**：Kafka/RocketMQ 双向消息支持
- **GraphQL**（可选）：支持复杂查询场景

#### 9.3.2 数据格式

- **结构化数据**：JSON（首选）/ Protobuf（性能场景）
- **时序数据**：InfluxDB Line Protocol / Prometheus 格式
- **文件交换**：PDF、CSV、JSON、Parquet

#### 9.3.3 错误处理

- **重试策略**：指数退避，默认 3 次
- **熔断**：连续失败自动熔断，避免雪崩
- **降级**：外部系统不可用时，平台降级为只读或本地缓存
- **告警**：所有集成异常均产生告警并通知管理员

#### 9.3.4 安全要求

- **传输加密**：TLS 1.3 强制
- **凭证管理**：所有外部系统凭证加密存储，由管理员管理，支持定期轮换
- **操作审计**：所有外部系统调用均记录审计日志
- **最小权限**：连接器账户按最小权限原则配置

### 9.4 集成开发与扩展

#### 9.4.1 内置连接器（出厂提供）

平台出厂时预置主流连接器模板（详见上 9.2），运营商按需启用与配置。

#### 9.4.2 自定义连接器

- **零代码配置型**：通过 Web 表单配置 HTTP API（URL、Header、参数映射）
- **脚本扩展型**：管理员上传 Python/JS 脚本，平台沙箱执行
- **SDK 开发型**：提供 Java/Python SDK，支持开发复杂连接器
- **审核发布**：所有自定义连接器需管理员审核

#### 9.4.3 连接器市场（可选）

管理员可上传连接器供单位复用，形成"连接器市场"。

### 10.2 安全要求

#### 10.2.1 认证与授权

- 支持账号密码 + MFA（多因素认证）
- 支持 SSO（OAuth 2.0 / SAML 2.0 / LDAP）
- 管理员强制 MFA
- 细粒度 RBAC 权限模型
- 最小权限原则

#### 10.2.2 数据安全

- 传输加密：TLS 1.3
- 存储加密：AES-256
- 敏感字段加密：单独加密存储
- 数据脱敏：日志、对外输出自动脱敏
- 数据分类分级：公开/内部/秘密/机密
- 数据不出网：所有推理在客户内网完成

#### 10.2.3 操作安全

- 高风险操作必须人工审批
- 审批过程留痕（谁、何时、何操作、结果）
- 操作前可模拟（试运行）
- 操作可回滚（撤销/补偿）
- 双人复核（可配置）适用于最高风险操作

#### 10.2.4 应用安全

- OWASP Top 10 防护
- SQL 注入、XSS、CSRF、SSRF 等防护
- 文件上传类型与大小校验
- API 限流、防爬
- 漏洞扫描与渗透测试定期执行

#### 10.2.5 基础设施安全

- 网络隔离：管理网/业务网/数据网分离
- 主机安全：基线加固、最小化部署、入侵检测
- 镜像安全：容器镜像签名、漏洞扫描
- 密钥管理：使用 HSM 或 KMS 存储敏感密钥

#### 10.2.6 合规

- 符合《数据安全法》《个人信息保护法》
- 符合卫星行业相关规定
- 满足等级保护三级要求（可按需提升）
- 支持国产化要求（CPU、OS、数据库、中间件）

### 10.3 可靠性要求

- **可用性**：99.9%（计划内停机除外）
- **容错**：单节点故障不影响整体服务
- **故障恢复**：RTO < 30 分钟，RPO < 5 分钟
- **数据备份**：每日全量备份 + 实时增量备份
- **异地容灾**：支持跨机房/跨地域部署
- **优雅降级**：外部系统不可用时，平台降级服务而非崩溃
- **熔断限流**：防止雪崩

### 10.4 可维护性要求

- **模块化**：微服务架构，单服务可独立升级
- **可观测性**：全链路 Trace、Metrics、Logging
- **监控告警**：服务健康、性能指标、错误率
- **日志**：结构化日志、统一日志平台
- **配置中心**：配置集中管理，支持热更新
- **CI/CD**：自动化构建、测试、部署
- **版本管理**：所有组件、模型、配置版本化
- **灰度发布**：支持功能灰度、模型灰度

### 10.5 兼容性要求

- **操作系统**：CentOS 7+、Ubuntu 20.04+、统信 UOS、麒麟
- **CPU 架构**：x86_64、ARM64
- **数据库**：PostgreSQL 14+、MySQL 8+、达梦、人大金仓、OceanBase
- **浏览器**：Chrome 100+、Edge 100+、Firefox 90+
- **国产化**：支持国产化软硬件栈
- **API 兼容**：保持 REST API 向后兼容

### 10.6 可扩展性要求

- **水平扩展**：所有微服务无状态，支持横向扩展
- **向量库**：支持集群扩展
- **模型推理**：支持多 GPU 并行、模型分布式
- **存储**：支持 PB 级数据存储
- **用户规模**：单实例支持 500 用户，集群支持 5000+ 用户
- **租户**：单租户起步，架构上预留多租户扩展能力

### 10.7 国际化与本地化

- **语言**：中文为主，预留英文扩展
- **时区**：支持多时区
- **日期格式**：本地化
- **多语 Prompt**：支持多语 Prompt 模板
- **字符集**：UTF-8 全面支持

### 10.8 易用性要求

- **帮助体系**：完善的帮助文档、视频教程
- **提示引导**：关键操作有引导提示

- **快捷键**：常用操作支持快捷键

- **部署方式**：支持 Kubernetes、Docker Compose、虚拟机
- **部署规模**：最小 4 节点起步（管理、控制、推理、数据）
